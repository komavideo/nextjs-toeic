## 0.1.0 リリース確認手順

次回リリースでも同じ確認を再現できるよう、公開判断前に次の順で確認します。コマンドは `web/` で実行し、結果はこの文書の該当セクションへ検証日時、実行コマンド、結果、確認観点として追記します。

### 自動検証

1. `pnpm validate:data` を実行し、問題データのスキーマ、ID、選択肢、正解参照、タグ、解説、`reviewed` フラグにエラーがないことを確認する。
2. `pnpm test` を実行し、単体テスト、問題データ検証の自己テスト、レビュー文書検証の自己テストが成功することを確認する。
3. `pnpm build` を実行し、問題データ検証、レビュー文書検証、Next.js の静的エクスポートが成功し、`web/out/` が生成されることを確認する。

### 問題レビュー証跡

1. `docs/QUESTION_REVIEW.md` の「リリース確認」と「0.1.0 問題データ検証実行記録」を確認し、フラット化後の設問数が300問以上であることを確認する。
2. Part 5 / 6 / 7 のレビュー記録で、対象設問がすべて `レビュー完了` になっていることを確認する。
3. `要修正` または `保留` の設問が残っていないこと、ETS 公式・市販教材・既存問題の再現と判断される設問が残っていないことを確認する。

### 手動 QA

1. 主要フローとして、初回ホームの「今日の学習ミッション」から5問クイックを開始し、回答、採点、解説、結果、ホーム復帰、進捗反映まで確認する。
2. 通常ホームの「今日の学習ミッション」で、復習、苦手 Part、苦手タグ、未回答優先（`unanswered=1`）の各導線が該当する演習へ遷移し、未回答優先では未回答問題が先頭に並び、不足時のみ回答済み問題で補完されることを確認する。
3. 復習フローとして、復習対象なしの表示、明日以降のみの復習予定一覧と復習開始ボタン非表示、期限到来データ投入後の復習一覧、復習開始、回答、SRS 更新、結果表示を確認する。直接 `/practice?mode=review` へアクセスした場合も、期限到来（期限切れ・今日）の問題だけが復習キューに入ることを確認する。
4. レスポンシブ確認として、360px 前後のスマートフォン幅、768px 前後のタブレット幅、1120px 以上のデスクトップ幅で、ホーム、演習、復習、進捗、設定の主要画面に横スクロール、本文とナビゲーションの重なり、固定アクションバーの干渉、空白画面、Next.js エラー overlay がないことを確認する。
5. A11y 確認として、主要操作のキーボード到達、フォーカスリング、モーダルの `role="dialog"` / `aria-modal` / フォーカストラップ / Escape 終了 / フォーカス復帰、必要に応じた Lighthouse Accessibility スコアを確認する。
6. 権利表記として、初回ホーム、通常ホーム、設定画面で ETS 非提携とオリジナル問題方針が表示され、TOEIC をアプリ名、ロゴ、主ブランドとして使っていないことを確認する。

### 静的ホスティング確認

1. `pnpm build` 後、`python3 -m http.server 4173 --directory out` などで `web/out/` をローカル静的サーバーとして配信する。
2. `http://localhost:4173/`、`/practice/`、`/review/`、`/progress/`、`/progress/tag/?tag=<既知タグ>`、`/settings/` に直接アクセスし、各ルートが HTTP 200 で表示されることを確認する。
3. 静的配信上のホームで「今日の学習ミッション」から `/practice/?mode=review`、`/practice/?mode=part&part=...&tag=...`、`/practice/?mode=part&part=...&unanswered=1` が空白画面や console error なしで開始できることを確認する。進捗画面の苦手タグから `/progress/tag/?tag=...` へ遷移し、「このタグを練習」からタグフィルター済み演習を開始できることも確認する。
4. 各主要ルートでブラウザリロードしても 404、空白画面、Next.js エラー overlay が出ないことを確認する。
5. ブラウザ console の warning/error が 0 件であること、ローカル静的アセット以外への外部リクエストが発生していないことを確認する。
6. 実ホスティング URL が確定している場合は、同じ観点を実 URL でも確認し、ローカル確認とは別行で記録する。

### ロールバック判断

- ホームが空白化する、ミッション CTA が遷移しない、未回答優先ミッションで未回答問題が残っているのに既回答問題が先頭に出る、または静的配信で console error が再現する場合はリリースを止めて直前版へ戻す。
- ホーム初期表示の JavaScript が問題本文・解説の全量取り込みによって明確に肥大化した場合は、問題データの軽量メタデータ化を優先してリリースを保留する。
