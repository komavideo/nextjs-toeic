# UI Design: 5分リーディングドリル

## 1. アートディレクション

通勤中に片手で使える、静かで実務的な学習ダッシュボード。余計な装飾を避け、問題文、選択肢、正誤、復習予定を素早く読み取れる密度と明瞭さを優先する。

## 2. トーン&マナー

- 学習アプリだが、ゲーム的演出ではなく業務ツール風の落ち着いた UI にする
- ユーザーが繰り返し開いても疲れない、白基調の高コントラスト画面にする
- スマートフォン縦持ちで主要操作が親指範囲に収まるようにする
- 「できた / できない」を過度に演出せず、次に解くべき問題へ誘導する
- TOEIC 表記は説明目的に限定し、ブランド表現には使わない

## 3. デザイントークン

### 3.1 カラーパレット

| トークン | 値 | 用途 |
| --- | --- | --- |
| `background` | `#F7F8FA` | アプリ全体の背景 |
| `surface` | `#FFFFFF` | 問題領域、パネル、モーダル |
| `surfaceSubtle` | `#F1F4F8` | セクション背景、選択肢ホバー |
| `textPrimary` | `#111827` | 本文、見出し |
| `textSecondary` | `#4B5563` | 補足、メタ情報 |
| `textMuted` | `#6B7280` | キャプション、非アクティブ |
| `border` | `#D8DEE8` | 通常の境界線 |
| `borderStrong` | `#AEB7C6` | フォーカス前の強調境界 |
| `primary` | `#176B87` | 主要ボタン、現在位置、重要リンク |
| `primaryHover` | `#0F556E` | 主要ボタンのホバー |
| `primarySoft` | `#E4F3F7` | 選択中背景、軽い強調 |
| `success` | `#168A4A` | 正解、改善傾向 |
| `successSoft` | `#E6F6EC` | 正解背景 |
| `warning` | `#B7791F` | 復習期限、注意 |
| `warningSoft` | `#FFF4D8` | 復習期限背景 |
| `danger` | `#C2413A` | 不正解、削除、リセット |
| `dangerSoft` | `#FCE8E7` | 不正解背景 |
| `focus` | `#2563EB` | キーボードフォーカス |

色は青緑を主アクセントにし、全体が単一色相にならないよう、成功・警告・危険のセマンティックカラーを明確に分ける。

### 3.2 タイポグラフィ

| 用途 | サイズ | 行間 | 太さ |
| --- | --- | --- | --- |
| 画面タイトル | `24px` | `32px` | `700` |
| セクション見出し | `18px` | `26px` | `700` |
| 問題文 | `18px` | `30px` | `500` |
| 本文 | `16px` | `26px` | `400` |
| UI ラベル | `14px` | `20px` | `600` |
| 補足テキスト | `13px` | `18px` | `400` |
| 小さなメタ情報 | `12px` | `16px` | `500` |

- フォントは `system-ui`, `-apple-system`, `BlinkMacSystemFont`, `"Segoe UI"`, `sans-serif`
- 英文問題文は読みやすさを優先し、字間は `0`
- ビューポート幅に応じたフォントサイズ変更は行わない
- 問題文と選択肢は、画面内で最も読みやすい階層にする

### 3.3 余白

| トークン | 値 | 用途 |
| --- | --- | --- |
| `space-1` | `4px` | アイコンとラベルの間 |
| `space-2` | `8px` | 小要素間 |
| `space-3` | `12px` | フォーム行、選択肢内余白 |
| `space-4` | `16px` | 標準余白 |
| `space-5` | `20px` | スマホ画面左右余白 |
| `space-6` | `24px` | セクション間 |
| `space-8` | `32px` | 画面内の大きな区切り |

### 3.4 角丸・線・影

| トークン | 値 | 用途 |
| --- | --- | --- |
| `radius-sm` | `4px` | 入力、チップ |
| `radius-md` | `6px` | ボタン、選択肢 |
| `radius-lg` | `8px` | パネル、モーダル |
| `borderWidth` | `1px` | 標準境界 |
| `shadowPanel` | `0 8px 24px rgba(17, 24, 39, 0.06)` | モーダル、固定フッター |

カードやパネルは最大 `8px` の角丸に留め、丸みを強くしすぎない。

### 3.5 レイアウト幅

| 領域 | 幅 |
| --- | --- |
| スマートフォン本文 | `100% - 40px` |
| 演習画面最大幅 | `720px` |
| ダッシュボード最大幅 | `1120px` |
| デスクトップサイドバー | `240px` |
| 下部固定アクション高さ | `72px` |

## 4. 主要コンポーネント

### 4.1 アプリシェル

- デスクトップでは左サイドバー + メイン領域
- タブレットでは左ナビを細くし、ラベルを維持する
- スマートフォンでは下部タブナビに切り替える
- ナビ項目は「ホーム」「演習」「復習」「進捗」「設定」
- 現在地は `primarySoft` 背景と `primary` の左線または上線で示す

### 4.2 ボタン

| 種類 | 用途 | 仕様 |
| --- | --- | --- |
| Primary | 次へ、回答する、開始する | `primary` 背景、白文字、44px 以上の高さ |
| Secondary | 戻る、別 Part を選ぶ | 白背景、`border` 線、`textPrimary` |
| Tertiary | 補助リンク | 背景なし、`primary` 文字 |
| Danger | データリセット | `danger` 背景または `danger` 文字 |

- スマートフォンの主要ボタンは画面下部に固定できる
- アイコンは lucide 相当の線アイコンを想定し、ラベルと組み合わせる
- 無効状態は `textMuted` と `surfaceSubtle`
- フォーカス時は `focus` のアウトラインを表示する

### 4.3 問題パネル

- 白背景、`border`、`radius-lg`
- 上部に Part、難易度、タグ、進捗を表示
- Part 6 / Part 7 はパッセージと設問を分け、スマートフォンでは縦積みにする
- 問題文は最小 `18px`、行間 `30px`
- 長文はスクロール領域に閉じ込めず、ページ全体で自然に読ませる

### 4.4 選択肢

- 1選択肢あたり高さ `52px` 以上
- 左に `A` / `B` / `C` / `D` の固定幅ラベル
- 未選択は白背景、選択中は `primarySoft` 背景と `primary` 境界
- 正解は `successSoft` 背景、不正解は `dangerSoft` 背景
- 結果表示時は正解の根拠を下に表示する

### 4.5 進捗サマリー

- 正答率、連続学習日数、今日の完了数、復習予定数を表示
- 復習予定数には内訳（期限切れ / 今日 / 明日以降）を併記する
- 数値は大きく、ラベルは小さく
- スマートフォンでは2列グリッド、デスクトップでは4列グリッド
- カードの乱用を避け、必要な指標だけに絞る

### 4.6 Part 別成績

- 横棒またはリスト形式で Part 5 / 6 / 7 の正答率を表示
- 苦手 Part は `warningSoft` で軽く強調
- タップすると該当 Part の演習開始へ遷移する

### 4.7 復習リスト

- 復習予定を期限切れ / 今日 / 明日以降の3グループに分類し、各件数と一覧を表示する
- 期限切れ、今日の順に上から並べ、明日以降は復習開始の対象に含めない
- 各項目に復習期限、Part、タグ、直近結果を表示する
- 復習予定が1件もない場合は空状態を表示し、通常演習へ誘導する

### 4.8 モーダル

- データリセット確認、セッション中断確認に使用
- 幅はスマートフォンで画面左右 `20px` 余白、デスクトップで最大 `480px`
- タイトル、短い説明、Primary / Secondary の2アクション
- 背景の暗幕は `rgba(17, 24, 39, 0.36)`

### 4.9 トースト

- 保存完了、リセット完了、保存失敗の通知に使用
- 画面下部、下部ナビより上に表示
- 自動消滅は3秒
- 重要な失敗はトーストだけでなく画面内にも表示する

## 5. Phase 1 全画面リスト

### 5.1 ホーム / ダッシュボード

役割:

- 今日の学習状況と次にやるべき行動を表示する

主要要素:

- 今日の学習数
- 全体正答率
- 連続学習日数
- 復習予定数
- 今日の学習ミッション（復習、苦手 Part、苦手タグ、未回答、初回クイック）
- Part 別成績
- 直近セッション履歴
- ETS 非提携・オリジナル問題の注記

遷移:

- 今日の学習ミッションから、復習、Part 指定、タグ指定、未回答優先、クイックの各演習へ直接遷移
- Part 別成績から Part 選択済み演習画面

### 5.2 Part 選択 / 演習開始

役割:

- ユーザーが解く Part とセッション種別を選ぶ

主要要素:

- Part 5 / Part 6 / Part 7 の選択リスト
- 各 Part の説明、目安時間、正答率
- 「おすすめ」表示
- 難易度フィルター
- タグフィルター
- 開始ボタン

遷移:

- Part 選択後、該当 Part の演習画面へ
- 戻る操作でホームへ

### 5.3 クイック演習中

役割:

- Part 5 は 3 / 5 / 10 問から選べる短時間学習を進める（既定 5 問。Part 6 / Part 7 はパッセージセット単位）

主要要素:

- セッション進捗 `1/選択問数`（Part 6 / Part 7 は `1/セット内設問数`）
- Part 表示
- 残り目安時間
- 問題パネル
- 選択肢
- 回答ボタン
- 中断ボタン

遷移:

- 回答後、解説表示へ
- 最終問題後、セッション結果へ
- 中断操作で確認モーダル

### 5.4 Part 5 問題画面

役割:

- 短文穴埋め問題を素早く解く

主要要素:

- 英文 sentence
- 設問 prompt
- 4択
- 難易度、タグ
- 回答ボタン

遷移:

- 回答後、Part 5 解説表示へ

### 5.5 Part 6 / Part 7 パッセージ問題画面

役割:

- パッセージを読み、関連設問に回答する

主要要素:

- パッセージタイトル
- パッセージ本文
- 設問リスト
- 現在の設問位置
- 4択
- パッセージへ戻るショートカット

遷移:

- 各設問回答後、解説表示へ
- セット完了後、セッション結果へ

### 5.6 解答・解説画面

役割:

- 正誤と根拠を確認し、次の問題へ進む

主要要素:

- 正解 / 不正解のステータス
- 正解選択肢
- ユーザーの選択
- 解説文
- 関連タグ
- 復習予定に入ったかどうか
- 次へボタン

遷移:

- 次へで次の問題
- 最終問題ならセッション結果へ

### 5.7 セッション結果画面

役割:

- 短時間学習の成果と次アクションを提示する

主要要素:

- 正答数
- 正答率
- 所要時間
- 誤答一覧
- 復習予定に追加された問題数
- 「復習する」ボタン
- 「もう1セット」ボタン
- 「ホームへ戻る」ボタン

遷移:

- 復習画面
- 同条件で次セッション
- ホーム

### 5.8 復習画面

役割:

- 簡易 SRS の期限到来問題を解く

主要要素:

- 復習予定件数（期限切れ / 今日 / 明日以降の内訳）
- 期限切れ / 今日 / 明日以降に分類した一覧
- Part / タグ / 期限の表示
- 復習開始ボタン（期限到来＝期限切れ・今日がある場合のみ表示）
- 復習予定なしの空状態

遷移:

- 復習開始で復習セッションへ
- 空状態から通常演習へ

### 5.9 進捗分析画面

役割:

- 学習傾向と苦手領域を確認する

主要要素:

- 全体正答率
- Part 別正答率
- タグ別苦手一覧
- 連続学習日数
- 直近7日の学習量
- 復習定着状況

遷移:

- 苦手 Part から Part 選択済み演習へ
- 苦手タグからタグフィルター済み演習へ

### 5.10 設定 / データ管理画面

役割:

- 端末内データの管理とアプリ情報を確認する

主要要素:

- バージョン `0.1.0`
- 表示言語 `日本語`
- 進捗データの保存先説明
- データリセットボタン
- ETS 非提携の注記
- オリジナル問題方針

遷移:

- データリセットで確認モーダル
- リセット完了後、ホームの初期状態へ

### 5.11 初回利用 / 空状態

役割:

- 進捗がない状態で次の操作を明確にする

主要要素:

- 短い導入文
- 「クイックを開始」ボタン
- Part 選択へのリンク
- 保存は端末内のみである説明

遷移:

- クイック演習
- Part 選択

### 5.12 エラー / 保存失敗状態

役割:

- `localStorage` が利用できない場合やデータ破損時に、ユーザーが状況を理解できるようにする

主要要素:

- エラー概要
- 再試行ボタン
- データ初期化ボタン
- 端末設定やプライベートブラウズへの注意

遷移:

- 再試行で保存処理を再実行
- 初期化で確認モーダル

## 6. レイアウト方針

### 6.1 スマートフォン

- 画面幅 `360px` から破綻しない
- 下部タブナビを採用し、主要画面へ1タップで移動できる
- 演習画面では主要アクションを下部固定にする
- 選択肢は縦積みで、指で押しやすい高さを確保する
- 長文画面ではパッセージと設問を縦方向に並べ、読書の流れを阻害しない

### 6.2 タブレット

- サイドナビとメイン領域の2カラム
- 演習画面は中央に最大幅を設け、横幅が広がりすぎないようにする
- Part 6 / 7 はパッセージと設問を横並びにしてもよいが、本文幅は読みやすさを優先する

### 6.3 デスクトップ

- 左サイドバー + メインコンテンツ
- ダッシュボードは指標、Part 別成績、履歴を横並びで配置
- 演習画面は中央寄せで、余白を広めに取る
- 過度なカードグリッドにせず、情報の優先順位で配置する

## 7. アクセシビリティ

- すべての操作要素はキーボードで到達可能にする
- フォーカスリングを消さない
- 正誤は色だけでなくテキストとアイコンで示す
- ボタンのタップ領域は最低 `44px`
- コントラスト比は通常テキストで 4.5:1 以上を目標にする
- モーダル表示時は背景操作を抑止し、フォーカスをモーダル内に閉じる
- `prefers-reduced-motion` に対応し、不要なアニメーションを抑える
- 問題文、選択肢、解説はスクリーンリーダーで自然に読める順序にする

## 8. Visual Companion 作成方針

- `docs/VISUAL-COMPANION.html` は本書の全画面リストをすべて掲載する
- 静的ギャラリー形式とし、画面ごとにスマートフォン幅のモックを中心に表示する
- PC / タブレットのレイアウト方針も一部サンプルで示す
- 画像生成はシングルスレッドで順次行い、業務ツール風のキービジュアル、空状態、学習モチーフなど補助アセットに限定する
- UI テキスト、ボタン、ナビ、問題文は HTML/CSS のコードネイティブで表現する
- 生成画像に重要な日本語 UI テキストを焼き込まない

## 9. 実装時の注意

- 依存関係は勝手に追加しない
- アイコンが必要な場合は、既存のアイコンライブラリが導入されてから使用する
- `localStorage` へ保存するデータはバージョン付きにする
- 画面内コメントやコードコメントを追加する場合は日本語にする
- TOEIC という語は説明目的に限定し、ブランド表現には使わない
- 公式問題や市販教材に似せた文面、固有表現、設問構成を避ける
