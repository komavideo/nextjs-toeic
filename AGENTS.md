# Repository Guidelines

## プロジェクト構成とモジュール配置

このリポジトリは、TOEIC リーディング学習アプリの Next.js 実装を `web/` に置いています。プロダクト要件や UI 方針は `docs/` を参照してください。

- `web/app/`: Next.js App Router のページとレイアウト。ルートは `app/<route>/page.tsx` に配置します。
- `web/components/`: 画面別または共通 UI コンポーネントの配置先です。
- `web/lib/`: 進捗、SRS、保存、問題バンク、検証などのドメインロジックを置きます。
- `web/types/`: 共有 TypeScript 型を置きます。
- `web/data/`: Part 5-7 の問題 JSON を管理します。
- `web/out/`: `next build` による静的エクスポート結果です。

## ビルド・テスト・開発コマンド

コマンドは原則 `web/` で実行します。

- `pnpm dev`: 開発サーバーを起動し、`http://localhost:3000` で確認します。
- `pnpm build`: Next.js をビルドし、静的サイトを `out/` に出力します。
- `pnpm test`: `node:test` による単体テストと、問題データ・レビュー文書検証スクリプトの自己テストを実行します。

## コーディングスタイルと命名規則

TypeScript は `strict` を前提にし、React/Next.js App Router の標準的な構成に従います。インデントは 2 スペース、文字列は既存コードに合わせてダブルクォート、文末はセミコロンありにします。React コンポーネントは PascalCase、関数や変数は camelCase、ルートディレクトリ名は小文字の意味名を使います。スタイルは Tailwind CSS v4 を優先し、グローバル CSS は必要最小限にします。

## テスト方針

テストは Node.js 組み込みの `node:test` を使い、対象コードの近くに `*.test.ts` または `*.test.tsx` として配置します。変更後は、変更分に直接関係する最小限の確認だけを実行し、静的エクスポートまで確認する必要がある場合は `pnpm build` を実行します。プロジェクト全体のテスト、単体テスト全件、UI テスト全件は、利用者が手動で明確に指示した場合のみ実行してください。新しい依存関係が必要な場合は、先に理由と代替案を説明してください。

## コミットと Pull Request

履歴に合わせ、コミットメッセージは `feat: ...`、`chore: ...` などの短い Conventional Commits 形式を推奨します。PR には目的、主な変更点、確認コマンド、関連 Issue を記載してください。UI 変更ではスクリーンショットまたは録画を添付し、仕様判断がある場合は `docs/` との対応も説明します。

## エージェント向け注意

変更は最小限に保ち、既存の構成と命名に合わせます。コード変更後は関連範囲の確認コマンドを先に実行し、生成物や依存関係を勝手に追加しないでください。
